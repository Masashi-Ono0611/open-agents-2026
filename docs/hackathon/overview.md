## What to Build

Projects should sit at the intersection of:

- **AI agents** — autonomous, goal-driven agents; multi-agent swarms; iNFT-powered agents (ERC-7857)
- **Onchain infrastructure** — execution layers, agent frameworks, tooling primitives deployed on decentralized networks
- **DeFi** — agents that swap, settle, or coordinate value onchain
- **Identity** — agent identity, discoverability, and coordination via naming systems (ENS)

Core expectation: working systems with real onchain execution, not demos.

## Judging Process

- **Round 1 (async):** All submitted projects reviewed asynchronously
- **Round 2 (live):** Top ~20% advance to live judging
- Same criteria across both rounds

**Judging criteria:**
1. Creativity
2. Functionality
3. Technical difficulty
4. Solution impact

---

## Submission Requirements

All projects must include:

1. Working demo — live or recorded video (2–4 minutes)
2. Public GitHub repository with README and setup instructions
3. Contract deployment addresses (where applicable)
4. Clear explanation of how sponsor protocol features were used
5. Team member names and contact info
6. Apply for up to **3 Partner Prize** tracks

---

## Team Rules

- Max **5 members** per team
- Each member must apply and stake individually (ETH staking required via Hacker Dashboard)
- All project work must be created during April 24 – May 3
- Public libraries and boilerplate are allowed; pre-existing project-specific code is not
